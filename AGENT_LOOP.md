# AGENT_LOOP: 継続的改善ループ（開発サイクル）

> juice を AI エージェントが自律的に改善するための **開発サイクルの手順書**。
> プロジェクト非依存の「ループの回し方」だけを扱う。プロジェクトの中身（概要・課題・状態）は
> **[PROJECT.md](PROJECT.md)** に分離してある。
>
> エージェントは毎サイクル、本書（手順）と [PROJECT.md](PROJECT.md)（状態）の両方を読んでから着手する。

| ドキュメント | 役割 |
|---|---|
| **AGENT_LOOP.md**（本書） | ループの手順・運用ルール・起動コマンド（プロジェクト非依存） |
| **[PROJECT.md](PROJECT.md)** | 何を作っているか／バックログ・課題／設計原則／スタック／直前の作業・作業中 |

---

## 改善ループ（Loop）

エージェントは以下の 5 ステップを 1 サイクルとして繰り返す。

1. **ASSESS（現状を評価）**
   - 本書と [PROJECT.md](PROJECT.md) を読む（特に「作業状態」と「バックログ」）。
   - 「作業中」があればそれを継続。無ければバックログから優先タスクを 1 つ選ぶ。
   - 実装可能かを判断する。
2. **PLAN（計画を立てる）**
   - 実装方針・影響範囲・テスト方法を決める。
3. **IMPLEMENT（実装する）**
   - コードとテストを書き、関連ドキュメントを更新する。
4. **VERIFY（検証する）**
   - `make check`（lint + test）を緑にする。
   - 可能な限り常に、サンプルデプロイの E2E 確認を行う
     （手順は [PROJECT.md](PROJECT.md) の「サンプルデプロイの E2E 確認」）。
5. **UPDATE（状態更新・引き継ぎ）**
   - 下記「運用ルール」に従い、PROJECT.md の「作業状態」を更新して commit する。

---

## 運用ルール（UPDATE & HANDOFF の手順）

ステップ 5（UPDATE）は **毎サイクル必ず次の順で実行する**。これにより「いま何が終わって・
次に何をするか」が常に PROJECT.md の「作業状態（直前の作業／作業中）」に表れ、
どのエージェントでも引き継げる。

1. **直前の作業を仕上げる。** IMPLEMENT / VERIFY を終える（`make check` 緑・E2E 確認）。
2. **ドキュメント更新・課題の書き出し。** 仕様変更を docs に反映し、気づいた課題・TODO を
   PROJECT.md のバックログ or 「作業中」に書き出す（忘れる前に文章化する）。
3. **作業中 → 直前の作業 へ移す。** これまで「作業中」だった項目を「直前の作業」に移し、
   **結果をまとめる**（何を変えたか・検証結果・関連 commit）。
4. **次の作業を計画して「作業中」に書く。** バックログから次の 1 つを選び、「作業中」に
   **概要（ゴール）とタスク（チェックリスト）／完了条件**を書き出す。
5. **ここで commit する。** 「完了した直前の作業 ＋ 次の作業の計画」を 1 コミットに残す
   （状態を書いてからコミットするのが順序。実装はまだ始めない）。
6. **作業中を参照して着手する。** 次サイクルの ASSESS は「作業中」を読むところから始め、
   そのタスクに取り掛かる。完了したら 1 に戻る。

> 要するに **「直前の作業＝直近に終えた成果」「作業中＝いま/次に取り組む計画」** の 2 つを
> （PROJECT.md 上で）常に最新に保ち、その更新を commit の単位にする。

## エージェント実行コマンド

このループを開始するには：

```bash
claude remote-control "AGENT_LOOP.md 読んで作業して"

claude "AGENT_LOOP.md 読んで作業して"
claude "AGENT_LOOP.md のループを 1 サイクル実行してください"

```
